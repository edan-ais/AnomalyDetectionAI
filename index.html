Performance Review Sync Flow
Data Synchronization Flow Steps
Trigger: Recurrence
Run twice daily at 8:00 AM and 8:00 PM
Action: Get PowerApps Records
Connect to PowerApps and retrieve all performance review records
Condition: Check for Changes
Compare PowerApps data with existing SharePoint records
Apply to Each: Process Records
Loop through each record that needs updating
Condition: Record Type
Determine if record is for manager view or employee view
Action: Create Security Copies
Generate the four security-specific versions of each record
SharePoint: Update or Create Items
Update existing SharePoint items or create new ones with appropriate permissions
Action: Set Item-Level Permissions
Apply specific read/write permissions to each record
Condition: Check for Errors
Verify all records were successfully processed
Action: Send Notification
Alert system administrator of any synchronization issues
Security Implementation

Due to SharePoint's security model, the flow creates four copies of each record:

Copy Purpose Who Can Edit Who Can View
Manager Working Copy Manager Manager Only
Manager Shared Copy Manager Manager + Employee
Employee Working Copy Employee Employee Only
Employee Shared Copy Employee Employee + Manager
